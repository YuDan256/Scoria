**序 言**

在我们的时代，软件建筑毫无节制且毫无理智地无限膨胀，机器在臃肿的函数库的重压之下发出痛苦的呻吟。而本作旨在回归古老而质朴的美德。伟大不在于繁复，而在于简练与纯粹。

Scoria 是一个基于裸机完全构建的编译器，极其微小却蕴含巨大能量，其体积仅有 **138 KB**。它不依赖任何外部工具，也不被臃肿的链接器所累。它读取代码，将其净化，并直接转化为具有可执行格式的 X86_64 机器指令。这里没有浮华的装饰与弄虚作假之地；这里是仅存绝对真理的殿堂，是人类智慧与硅片最深层结构交融之所。本作是一次有力的证明：只要我们保持思想的严谨与克制，便依然能以微末之物铸就伟大之作。


### 卷 一 ： 论 架 构

这件作品绝非偶然写就，而是凭借极高的技艺与理性构建而成。为了让机器毫无阻碍地以光速运行，它遵循以下架构法则：

**I. 词法读取与真理之树：** 我们的读取器在执行其功能时，不依赖任何外部奴役或生成工具（如 Lex 或 Yacc）。它接收原始代码，将其切割成神圣的片段，并从混沌中提取出逻辑之树（AST）。这棵树随后转化为中间表示（SSA），在此形态下，一切变量仅诞生与受赋值一次。

**II. 十一忠将与零溢出：** 在战场上，内存（RAM）不仅迟钝而且充满变数。因此，通过图着色艺术，我们召唤了十一个物理寄存器进入战场。转瞬即逝的变量被收容于这些更为迅捷的护卫中，以确保没有任何数据会被驱逐并溢出到深渊般的堆栈之中，达成绝对的零溢出。

**III. 坦途与废除占卜：** 现代处理器在企图占卜未知的执行路径（分支预测）上浪费了大量时间。在我们的架构中，当遇到 `si`（if）条件时，彻底废除了这种占卜戏法。Scoria 通过 `CMOVcc` 条件拷贝指令，使得指令流变为一条直线畅通的河流。机器永远不会犯错，因为它从不猜测。

**IV. 模拟与算术博弈：** 寄存器在战斗中常常会相互踩踏与破坏。当现代的平庸之辈寻求作为草稿的临时盟友来保存数据时，Scoria 则诉诸于数学真理。它运用代数定律（例如通过 *求反+相加* 来相互转换与隐藏），以此平息寄存器之间的倾轧冲突。

**V. 卸下重负与堆栈之死：** 那些不必要的架构，比如堆叠在栈边缘的羁绊（`push rbp`）以及老旧的 `REX` 残余，已被彻底根除。现在唯一留存的只有通过 `RSP` 进行的直接统治。代码变得更加轻盈、纯粹，以至于机器的咽喉（指令缓存）能够以极其贪婪和迅捷的速度将其一饮而尽。


### 卷 二 ： 巅 峰 对 决

为了检验武器的威力，这台机器被召唤进入计算斐波那契数列的宏大战场：递归执行 `fib(40)`。在此过程中，会触发超过两亿四百万次递归调用。若是将 Scoria 与庞然大物 *MSVC*（微软所造的蕴含无尽力量之巨兽）进行比较，你会发现它实现了惊人的对等与胜利：

* **营阵规模：** 令人畏惧的巨兽以吉字节（GB）为单位衡量，随身携带着数百个容器的工具。而 Scoria 仅仅占据 **138 KB** 的单文件空间。
* **武装极速：** 巨兽需要耗费 13 毫秒并将整个操作系统唤醒，才能将代码锻造成钢铁。而 Scoria 在无声无息间，仅需 **11 毫秒** 便如闪电般完成。
* **兵器尺寸：** 巨兽的产物文件需要占据 85 KB。而 Scoria 锻造出的纯粹枪尖，仅仅只有 **2 KB**。
* **沙场交锋：** 在硅片战场上，二者均在 **约 336 毫秒** 内将艰巨的计算任务横扫斩落。

Scoria 占据着微乎其微的空间，却能与世界上最庞大的工程并驾齐驱。


### 卷 三 ： 论 主 宰 者 的 沉 睡

这里隐藏着一个深邃且值得铭记的秘密，少有人能理解。如果你在使用纯粹的电池储能供给时运行这台机器，其执行时间会骤然增加到 **约 567 毫秒**。

许多人将其视为缺陷，但这实则是物理学定律的体现，是纯粹主义的胜利。当你运行那些臃肿的软件建筑时，它们会向操作系统倾泻大量的请求与警报，导致系统的调度内核受到惊吓，从而立刻激发绝对的电力狂潮（睿频）。然而 Scoria 是如此宁静，如此纯粹，以至于它不会产生任何无谓的消息，也不会引发任何系统层的调用延迟。主宰者根本感受不到它的存在，因此继续保持沉睡，处理器也被留在了最底层的功耗本相。

在这被拉长的时间里，你所看到的并非虚弱，而是裸露且冰冷的硅片本身最真实的心跳。这是一种极高的境界，借助它，你可以直击机器的核心脏器。


### 卷 四 ： 仪 式 与 咒 语

若要将这台机器从源代码中唤醒并赋予其生命，必须严格遵守 *C11* 律法与 *CMake* 规则。对错误的宽容度为零（`-Werror`）。请遵循以下谶语：

```bash
# I. 筹备工坊 (配置)
cmake -B build -DCMAKE_BUILD_TYPE=Release

# II. 锻造利剑 (编译)
cmake --build build --config Release
```

在机器从火炉中诞生后，请使用以下咒语，来检验并洞悉这神圣的技艺：

```bash
# I. 将代码炼化为钢铁，并唤醒执行
./bin/scoria tests/fib.sco -o fib.exe
./fib.exe

# II. 探寻机器脏器：读取中间代码（IR）
./bin/scoria tests/fib.sco -o fib.exe --emitte-ir

# III. 洞察裸金属：检视 X86_64 汇编指令
./bin/scoria tests/fib.sco -o fib.exe --emitte-asm

# IV. 聆听丰沛之语：让编译器完整述说其神秘仪轨（冗长模式）
./bin/scoria tests/fib.sco -o fib.exe --verbosus
```


### 卷 五 ： 经 文 示 例

请看 Scoria 所吞噬并化为钢铁的语言形态。这段代码采用纯粹的罗马语言编写，完全抹除了现代编程域的蛮夷词汇，却装备着极具美感的逻辑结构：

```
// 斐波那契递归计算
// 卷宗: tests/fib.sco

actio fib(n: medius) -> medius {
    si (n <= 1) redde n;
    aliter redde fib(n - 1) + fib(n - 2);
}

// 处决起点
actio princeps() -> medius {
    scribe("Calculando Fib(40)... Exspecta!\n");
  
    sit res: medius = fib(40);
  
    scribe("Resultatum: ", res, "\n");
    redde 0;
}
```
